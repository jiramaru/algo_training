Exercice 1 - Problèmes de programmation (Logique & Algorithmes)

=== Problème 1 : Nombres divisibles (Facile) ===
Objectif : Parcourir les nombres de 1 à N et afficher une valeur selon les diviseurs.

Entrée : N A B mot1 mot2 (exemple : 10 2 3 pair triple)
- N : le nombre maximum (inclus)
- A : premier diviseur
- B : deuxième diviseur  
- mot1 : texte à afficher si divisible par A
- mot2 : texte à afficher si divisible par B

Sortie : Pour chaque nombre de 1 à N :
- Si divisible par A et B : afficher mot1+mot2
- Si divisible par A seulement : afficher mot1
- Si divisible par B seulement : afficher mot2
- Sinon : afficher le nombre

Exemple :
Entrée : 10 2 3 pair triple
Sortie : 1 pair triple pair 5 pair triple pair 9 pairtriple

Entrée : 10 3 5 trip cinquo
Sortie : 1 2 trip 4 cinquo trip 7 8 trip cinquo

=== Problème 2 : Vérificateur de palindrome (Facile) ===
Objectif : Vérifier si une ligne de texte est un palindrome (se lit identiquement dans les deux sens).

Règles :
- Ignorer les espaces et la ponctuation
- Ignorer la casse (majuscules/minuscules)
- Considérer uniquement les lettres et chiffres

Entrée : Une ligne de texte (UTF-8)
Sortie : OUI (si palindrome) ou NON (si non palindrome)

Exemple :
Entrée : "Esope reste ici et se repose"
Sortie : OUI

=== Problème 3 : Élément le plus fréquent (Facile à Moyen) ===
Objectif : Trouver l'élément qui apparaît le plus souvent dans une liste et compter combien de fois.

Entrée : Premier nombre n (nombre d'éléments), suivi de n éléments
- Les éléments peuvent être des nombres ou du texte
- n peut être 0 (liste vide)

Sortie : Deux valeurs : élément fréquence
- Si plusieurs éléments ont la même fréquence maximale, renvoyer le premier rencontré
- Si la liste est vide, afficher : "vide 0"

Exemple :
Entrée : 5 chat chien chat oiseau chat
Sortie : chat 3
(Explication : "chat" apparaît 3 fois, c'est le plus fréquent)
